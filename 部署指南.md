# Quick-Trans 部署指南

## 腾讯云 CloudBase 静态网站托管部署步骤

### 方法一：通过控制台手动部署（推荐初次使用）

#### 1. 创建 CloudBase 环境

1. **登录腾讯云控制台**
   - 访问：https://console.cloud.tencent.com/
   - 登录您的腾讯云账号

2. **开通 CloudBase 服务**
   - 在搜索框搜索"CloudBase"或"云开发"
   - 点击"立即使用"

3. **创建环境**
   - 点击"新建环境"
   - 填写环境信息：
     - **环境名称**：`quick-trans`（或其他您喜欢的名称）
     - **计费模式**：按量付费（有免费额度）
     - **地域**：选择离您最近的地区（推荐：广州、上海、北京）
   - 点击"立即开通"
   - 等待环境创建完成（约1-2分钟）

4. **开通静态网站托管**
   - 进入创建好的环境
   - 在左侧菜单找到"静态网站托管"
   - 点击"开通"
   - 选择"免费版"（通常足够个人项目使用）

#### 2. 上传文件

1. **构建项目**
   ```bash
   cd d:/24孔板转染计算器
   npm run build
   ```

2. **上传文件到 CloudBase**
   - 在 CloudBase 控制台的"静态网站托管"页面
   - 点击"文件管理"
   - 点击"上传文件夹"
   - 选择项目目录下的 `dist` 文件夹
   - 等待上传完成

3. **配置访问地址**
   - 上传完成后，CloudBase 会提供一个默认的访问地址
   - 格式类似：`https://your-env-id.service.tcloudbase.com`
   - 您可以记下这个地址，用于在手机或其他设备上访问

#### 3. 测试访问

- 在浏览器中打开 CloudBase 提供的访问地址
- 在手机浏览器中打开同一个地址
- 确认功能正常

---

### 方法二：使用 CLI 工具部署（推荐后续更新）

#### 1. 安装 CloudBase CLI

```bash
npm install -g @cloudbase/cli
```

#### 2. 配置环境 ID

1. 在 CloudBase 控制台获取您的环境 ID
   - 进入"设置" → "环境设置"
   - 找到"环境 ID"并复制

2. 编辑项目根目录下的 `cloudbaserc.json` 文件
   ```json
   {
     "version": "2.0",
     "envId": "your-env-id",  // 替换为您的实际环境 ID
     ...
   }
   ```

#### 3. 登录 CloudBase

```bash
cd d:/24孔板转染计算器
cloudbase login
```

会打开浏览器进行授权登录。

#### 4. 部署

```bash
npm run deploy
```

或者：

```bash
cloudbase framework deploy
```

## 部署后管理

### 更新项目

当您修改代码后，需要重新部署：

1. **构建项目**
   ```bash
   npm run build
   ```

2. **重新部署**
   - 方法一：在控制台的"文件管理"中删除旧文件，重新上传 `dist` 文件夹
   - 方法二：使用 CLI 命令 `npm run deploy`

### 绑定自定义域名（可选）

如果您有自己的域名，可以绑定到 CloudBase：

1. 在"静态网站托管"页面点击"设置"
2. 点击"域名配置"
3. 点击"添加域名"
4. 按照提示配置 DNS 解析
5. 等待 SSL 证书自动签发（支持 HTTPS）

### 查看访问统计

在 CloudBase 控制台可以查看：
- 访问次数
- 流量使用情况
- 错误日志等

## 常见问题

### Q1: 如何获取访问地址？
A: 在 CloudBase 控制台的"静态网站托管"页面可以看到默认访问地址。

### Q2: 可以在手机上使用吗？
A: 可以！部署后，任何人都可以通过访问地址在手机或其他设备上使用。

### Q3: 如何确保数据安全？
A: 这是一个纯前端应用，所有数据都在浏览器本地处理，不会上传到服务器，非常安全。

### Q4: 更新项目需要多长时间？
A: 通常构建需要几秒钟，上传文件需要几分钟，整体流程很快。

### Q5: 可以多台设备同时使用吗？
A: 可以，每台设备独立运行，互不影响。
